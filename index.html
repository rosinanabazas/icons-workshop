Web Icons

If you're building an icon system for a website, you have some options. You can choose between inline PNG images and CSS sprites. But also, if the icons will be vector images (much more common these days), you have 2 more options: using inline SVG and using icon fonts.

Let's compare.

PNG/JPG Icons

This is the less recommended way to add icons because they are raster pixel images so they won't be scalable. You won't be able to change it's color by css and they'll have more weight. Also they won't look good on some devices.

For using them you just need to put the file on the img folder and call it from the html or as a background-image on the css.

Example:

<a href="www.twitter.com"><img src="img/twitter-logo.png"></a>

CSS Sprites

Sprites are a means of combining multiple images into a single image file for use on a website, to help with performance and reducing the number of HTTP requests made for image resources. It's about getting the image once, and shift it around and only display parts of it. This reduces the overhead of having to fetch multiple images.

Sprite generators:

  • SpitePad
  • SpriteGen
  • SpiteBox

Example:

<a href="www.twitter.com"><img src="img/twitter-logo.png"></a> Test

SVG Icons

SVG is an image format for vector graphics. It literally means Scalable Vector Graphics. The interesting part here is that you can either add the svg file same as an image, or drop that code right into an HTML document and the SVG image will show up just the same as if you put it in an img. To get this code you can open the svg file on your text editor and copy it.

  • You have same CSS control as with a font.
  • Scalable.
  • Looks fine on all devices.

Example:

<a href="www.dribbble.com"><img src="img/dribbble-logo.svg"></a>

Font Icons

They're also vector graphics, which means they scale up perfectly and look awesome on retina. And just like with traditional typography, you can easily change an icon's size, color, drop shadow, or anything else that's possible with CSS.

You can either use a pre-generated font or create your own custom font.

  • Scale easily
  • Change color easily
  • Include shadows easily
  • Have good browser support in general
  • Can be designed on the fly (by making changes on :hover, etc.)
  • Can do anything image icons can do (change opacity, rotate, etc.)
  • Have smaller file sizes since they contain fewer characters than full typeface

Font Awesome

How it works:

  1. Go to Font Awesome and download the zip file
  2. Extract it and copy the entire font-awesome directory into your project.
  3. In the < head > of your html, reference the location to your font-awesome.min.css.
  4. You can place Font Awesome icons just about anywhere using the CSS Prefix fa and the icon's name. It's designed to be used with inline elements < i > or < span >
  5. To increase icon sizes relative to their container, use the fa-lg, fa-2x, fa-3x, fa-4x, or fa-5x classes.
  6. You can also simply change the icon's size and color changing it's font-size and color on the css
  7. There are also some animated icons: Loading...

Example:

<i><class="fa fa-diamond fa-lg"></i>
fa-lg fa-2x fa-3x fa-4x fa-5x

Icomoon

On Icomoon you can download a predesigned icon pack or build a new one.

How to create a new font icon:

  1. Go to Icomoon and click on Import icons button at the top left and select the icons you need.
  2. Select them and click on "Generate font" link at the bottom.
  3. Download the font and extract it.
  4. Copy the Fonts folder into your project.
  5. Copy the style.css file on the stylesheet folder.
  6. In the < head > of your html, reference the location to your Icomoon stylesheet: style.css
  7. Save the selection.json file somewhere so you can import it again on the app if you need to make further additions/modifications.

Example:

<span><class="icn-clock"></span>
